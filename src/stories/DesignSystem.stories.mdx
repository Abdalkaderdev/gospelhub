# GospelHub Design System

A comprehensive design system built with TypeScript and styled-components for the GospelHub Bible reader application.

## Philosophy

Our design system is built around the principles of:
- **Consistency**: Unified visual language across all components
- **Flexibility**: Compound components that can be composed in multiple ways
- **Accessibility**: WCAG compliant components with proper semantic markup
- **Performance**: Optimized animations and efficient rendering

## Component Architecture

### Compound Components
We use the compound component pattern for maximum flexibility:

```tsx
<BibleReader verses={verses} currentBook="Genesis" currentChapter={1}>
  <BibleReader.Header>
    <Text variant="caption">King James Version</Text>
  </BibleReader.Header>
  <BibleReader.Content>
    {verses.map((verse, index) => (
      <BibleReader.Verse key={verse.verse} verse={verse} index={index} />
    ))}
  </BibleReader.Content>
</BibleReader>
```

### Custom Hooks
Reusable logic extracted into custom hooks:

- `useBibleData` - Bible data management and search
- `useUserPreferences` - User settings and preferences
- `useAnimations` - Animation variants and controls

## Getting Started

```tsx
import { ThemeProvider } from 'styled-components';
import { designTheme, Button, Card } from './design-system';

function App() {
  return (
    <ThemeProvider theme={designTheme}>
      <Card>
        <Button variant="primary">Get Started</Button>
      </Card>
    </ThemeProvider>
  );
}
```
